# Contributing

coreartifact is personal-first while it is pre-1.0: the design is still
settling, and **code contributions are not being accepted before 1.0**.
Pull requests will be closed with thanks, not reviewed.

**Issues are welcome** — bug reports with reproduction steps are
especially valuable, and honest-absence bugs (a facet showing a value it
cannot actually verify) are treated as the most serious class. There is
no response-time commitment.

The repository's full history is public by design, including the
in-progress specs, issue packets, and retros under `docs/` — reading
`docs/spec-v1.md` first will answer most "why is it like this?"
questions.
