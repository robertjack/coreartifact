# Core contracts III: attribution

Split out of ISS-0001 on 2026-07-14 after it escalated twice. Attribution is
one small pure function — and it killed both ladders, because git has more ways
to lay out a repository than anyone remembers. It gets its own issue and its own
focused edge-case suite.

Prior art: `escalated/ISS-0001-attempt-2` has an implementation that handles
the worktree, main-checkout, non-git and symlink cases. It still fails the
submodule / `--separate-git-dir` case and the `GIT_DIR` leakage case. Start from
it.

## Why this function is load-bearing

`resolveAttribution` decides **which repo's spool a session's events are
appended to**. It is called by the hook artifact at capture time (to find the
spool) and by ingest (to fill `sessions.repo_root` / `worktree_path`). Getting
it wrong does not produce a bad display — it writes **ground truth into the
wrong place**, or into a place git will overwrite. There is no recovering from
that after the fact.

It must be expressible with **no package dependency**: pure `git rev-parse`
shell-outs plus path logic, because the zero-dependency hook artifact calls the
same logic.

## The contract

Given a cwd and an init-root fallback:

- cwd inside a linked **worktree** → repo root is the **main** repo root; the
  worktree checkout path is recorded alongside it.
- cwd inside a **main checkout** → that root; worktree path ABSENT.
- **non-git** cwd → the supplied init-root fallback; worktree path ABSENT.

## The mechanism (do not re-derive — three reviews established this)

**Compare `git rev-parse --git-dir` with `--git-common-dir`.**

- **Equal → a main checkout.** repo_root = `--show-toplevel`, worktree path
  ABSENT — *no matter where the git dir physically lives*.
- **Different → a genuine linked worktree.** Derive the main root from
  `git worktree list --porcelain`'s first entry.

**Never infer linkage from `dirname(gitCommonDir)`.** That assumes the git dir
is always `<repo_root>/.git`, which is false for a **submodule** (git dir lives
at `<super>/.git/modules/<name>`) and for `git init --separate-git-dir`. Attempt
1 did this, and a submodule checkout resolved to a `repo_root` **inside
`.git`** — meaning the append-only spool, the one artifact the ledger cannot
regenerate, would be written into a directory git owns and rewrites.

**Realpath both sides before comparing.** `--show-toplevel` returns a physical
path while `--git-common-dir` is relative to the caller's logical cwd. On any
symlinked path (macOS `/var` → `/private/var`, `$TMPDIR`) an unrealpathed
comparison fabricates a worktree path for a main checkout and mints **two
`repo_root` identities for one repo** — which means two registry entries and two
ledgers for the same repository.

**Neutralize the ambient git environment.** `GIT_DIR` / `GIT_WORK_TREE` in the
caller's environment redirect `git rev-parse` at another repository entirely
(reviewer S2, attempt 2). A session running with a stray `GIT_DIR` would have
its spool written into an unrelated repo. Scrub those variables from the
environment of the git invocations this module makes; resolution must depend on
the **cwd** and nothing else.

## Invariants (contract prose, not acceptance)

- No package dependency: `git` shell-outs and path logic only.
- Pure: no writes, no network, no ledger access.
- A resolution failure returns the init-root fallback — it never throws and
  never fabricates a path.

## Test-harness contract

Unit tests only (plain vitest) at `tests/unit/core/attribution.test.ts`, against
**real git repositories** created in a tmpdir — no mocked git, ever. This
function's entire difficulty is real git's behavior; a mocked test would assert
the author's assumptions back at them, which is exactly how both ladders died.

Build each fixture repo for real: `git init`; `git worktree add` for the
worktree case; a real submodule (`git submodule add` of a second local repo) and
a `git init --separate-git-dir` checkout for the git-dir cases; a symlinked path
to a main checkout for the identity case; a plain tmpdir for the non-git case;
and a case invoked with `GIT_DIR` set to a foreign repo for the environment
case.

An acceptance harness exists at HEAD (`tests/acceptance/`, from ISS-0002); this
issue does not add to it. `pnpm test` must be green when you finish.
