# Global operator state — append-only fold log

**Read `docs/gotchas.md` first** — entry 3 (allowlist env in tests), entry 4
(a concurrency test over a synchronous body is not a test), entry 5 (the
degradation law).

A fourth global artifact under `~/.coreartifact/` (file name
`state.jsonl`), holding the machine's install id, ping consent, and
last-ping time. It is the **exact append-and-fold pattern the registry
already uses** — `src/core/registry.ts` is the normative prior art; mirror
its shape, totality discipline and test style. The registry rewrite of
2026-07-14 deleted the system's only read-modify-write and with it a whole
bug class; this surface must not reintroduce it. Consumers: the consent
question at init and the weekly ping (a later issue); doctor never reads it;
per-repo uninstall never touches it.

## The log

```
{"v":1,"op":"install","install_id":"<id>","at":"<iso8601>"}
{"v":1,"op":"consent","ping":true,"at":"<iso8601>"}
{"v":1,"op":"ping","at":"<iso8601>"}
```

- Every state change is **one atomic O_APPEND of one line**. No read, no
  lock, no read-modify-write anywhere. Writers: `install` appended once at
  first init on the machine; `consent` when the opt-in answer is recorded;
  `ping` once per send attempt.
- `v: 1` is a durable per-line version contract (the registry's
  discipline) — the format evolves by adding ops, never by migration.
- Compaction is a v1.1 concern; the file grows only on rare state changes.

## The fold

`readState` folds the log into `{install_id, consent, last_ping_at}` plus a
skipped-line count, and is **total** — a damaged state file must never take
down a CLI command:

- `install_id` = the **first** `install` op's id. First-wins is load-bearing:
  the install id is generated once and is the only value that ever leaves
  the machine (in the ping), so it must never silently change. A second
  `install` line (crash race at first init) is folded away, not an error.
- `consent` = the **last** `consent` op's `ping` value — last-write-wins for
  a mutable setting. **Default with no consent op ever recorded = `false`.**
  This is the degradation law applied to consent: silence folds to "no",
  never to a flattering "yes"; nothing pings without a positive record.
- `last_ping_at` = the latest `ping` op's `at`.
- Missing file → empty state (no install id, consent false, no ping time).
  Corrupt line, truncated line, non-object JSON, wrong `v`, unrecognized
  `op`, or a shape-invalid op (an `install` with no `install_id`) → skipped
  and counted, never thrown, never partially folded. Fabricate nothing: a
  missing `at` never becomes an empty-string timestamp.

Idempotence is a property of the fold, not the write (the registry ruling):
duplicate lines change nothing about the folded state.

## Paths

The state file resolves under the same global root the paths module already
exposes for the registry, including the same environment-variable override —
one override isolates a test subprocess's entire global surface. Extend the
paths module with the state-file location; do not invent a second override.

## Invariants (contract prose, not acceptance)

- This module performs no network I/O and no prompting — it is storage only.
- Nothing here runs on the hook artifact's path; capture knows nothing of
  operator state.
- Uninstall (a later issue) removes per-repo artifacts and registry entries
  only; this file is machine-scoped and survives.

## Test-harness contract

Unit tests only, in a tmpdir global root via the paths override. Mirror the
registry test file's structure (it is the house pattern for fold totality:
hostile shapes fed one by one, skip-and-count asserted, no throw). The
concurrency test must spawn **N separate processes** appending distinct ops
(gotchas entry 4 — Promise.all over a synchronous body runs serially and
cannot detect a lost line) and assert all N lines survive and the fold
reflects them; build child env from scratch per gotchas entry 3.
