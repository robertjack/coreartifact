# log: one line per session, unioned across repos, honest about gaps

**Before writing anything, read `docs/gotchas.md`** — the bugs prior slices already bled on (entry guards, newline ordering, denylist env, tests that can't fail, the degradation law, verify-on-the-floor). Reusing those fixes is cheaper than rediscovering them.


`log` is the first thing a user sees working. It reads and prints, and never
writes anything but the ledger the ingest slice already owns — but it is `med`
risk, not low (plan-gate retag): it owns the **absent-marker module that the
show slice consumes as a contract**, and it touches the ingest and registry
surfaces other issues own. It is also the slice where the **degradation law
becomes visible**.

## What log does

1. Read the **registry** (the global list of ledger roots).
2. For each registered ledger: **ingest** its spool (the ingest slice owns the
   mechanism; `log` is its trigger), then read back the sessions.
3. Print **one line per session**, unioned across every registered repo,
   carrying at minimum: **short id, repo, status, kind-or-absent, start time,
   command count, footprint count**.
4. Print the ingest report's warnings and skipped lines.

Command count comes from the events table (Bash tool events for the session);
footprint count from the footprint table. Neither is re-derived from the spool.

## The absent marker (shared surface)

This issue owns the **rendering seam for absence**, which both `log` and `show`
use. Every ABSENT facet renders as an **explicit absent marker**, visually and
textually distinguishable from empty, zero and success.

**`kind` is NOT the absent case (2026-07-14 amendment).** The recording pass
found the signal — `SessionStart` carries a `model` key iff interactive
(`docs/recording-pass.md` finding 3) — so ingest populates `kind` as
`interactive` or `headless` for every session that has a SessionStart line.
`kind` reads ABSENT only as a **drift fallback** (no SessionStart captured, or
a future release drops the key). Render all three states distinctly: an absent
kind must never print as a blank column, a dash that also means "zero", or a
guessed `interactive`.

The **sha** facets are the reliable common path for the absent marker (the
SIGKILL stream has no SessionEnd, so `sha_after` is ABSENT), alongside
outcome-absent for backgrounded commands. Pick one unmistakable token, export
it from one module, and let `show` import it. A test asserting `log` output must
be able to tell "no commands ran" (count 0) from "we don't know" (ABSENT) by
reading the line.

## The worktree gap warning

Ingest warns, naming **any worktree that is missing the settings file** — i.e. a
worktree Claude Code sessions would run in silently uncaptured, typically a
hand-made `git worktree add` with no session involved (the accepted residual of
the layered-propagation ruling). It stays **silent** when propagation is
complete: no warning, no noise, nothing printed on the happy path. A warning
that always fires is a warning nobody reads.

Enumerate the repo's worktrees from git and check each checkout for the
per-repo settings file. That is the entire diagnostic.

**This is not the doctor.** No other diagnostics ship this campaign — no
checks, no health output, no suggestions beyond naming the gap. The doctor is
PRD-0002.

## Touching the log command (routed amendment)

The `log` command file was created by the ingest slice, which prints a minimal
session line and the ingest report. This issue **touches** it to swap in the
renderer and add the cross-repo union and the warning. That is a deliberately
routed shared-surface amendment, not a drive-by.

The ingest slice's acceptance asserts that a corrupt line is **named in
output**. Preserve that: the renderer must keep the skipped-line numbers in the
printed report. Its tests assert the line number appears, not its exact
formatting — so reformatting is safe, dropping the number is not.

## Invariants (contract prose, not acceptance)

- `log` is read-only with respect to the spool, and writes to the ledger only
  through the ingest module.
- No transcript is read. No cost or token facet exists this campaign.
- No network call, no telemetry ping.
- No tree or nested rendering: the timeline is flat. Nesting keys are captured
  but not rendered (v2).

## Test-harness contract

Copy the `tests/acceptance/` harness from the harness issue **verbatim** —
tmpdir-repo factory (with its **isolated registry**, which the two-repo union
test cannot work without), CLI runner, fixture replayer, worktree helper, and
its ledger readers.

Acceptance tests at `tests/acceptance/log.test.ts`:

- **One line per session with the required fields** — `init`, replay the
  `headless` stream through the installed hook, run `log`, assert the line
  carries short id, repo, status, kind-or-absent, start time, command count and
  footprint count. Assert the command count matches the number of Bash tool
  events in the fixture and the footprint count matches its distinct edited
  paths (both are properties of the fixture; read them from the fixture, do not
  hardcode a number that a re-recording would invalidate).
- **Two-repo union** — build two tmpdir repos sharing the harness's isolated
  registry, `init` both, replay a stream in each, run `log` from either, assert
  both repos' sessions appear.
- **Warning fires** — `init` a repo, then `git worktree add` a new checkout with
  the harness helper **without** replaying a WorktreeCreate event (simulating a
  hand-made worktree), run `log`, assert the warning names that worktree path.
- **Warning stays silent** — the propagated case (worktree created before
  `init`, or via a replayed WorktreeCreate) produces no warning at all.

Fixtures come from the fixtures issue's loader by scenario name.
