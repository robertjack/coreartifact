# init installs a coreartifact SKILL.md — agents self-verify by default

**Why (post-launch, 2026-07-21, operator-ruled):** aeh PR #7 made this
repo's own workers wrap gates in `cart check` and land VERIFIED. Every
other Claude Code user still has to know the convention. Claude Code
auto-discovers `.claude/skills/*/SKILL.md`; `init` already writes into
`.claude/`. Installing a small conventions skill at init hands the
self-verifying-delegation property to every repo that installs
coreartifact — no prompt edits, no harness changes.

## Rulings (binding)

1. **No consent question; loud install + `--no-skill` opt-out.** The
   skill is local-only guidance — nothing leaves the machine — so it is
   NOT the ping's class. Install by default (TTY and headless alike),
   always print the path installed, honor `--no-skill`. The operator
   deletes or opts out freely.
2. **The inversion law extends to it.** The skill joins the install
   backup; `uninstall` removes exactly what init installed and never
   touches user-authored skills. The ISS-0022 byte-identical tree
   comparison is the oracle.
3. **Drift is doctor's job.** The canonical text ships with the package
   (a versioned source module, resolved the way `hookArtifactSource`
   resolves the hook artifact — works from the packed dist). An
   installed copy differing from the running package's canonical text
   is a named doctor finding, never silently rewritten.
4. **Gitignored by default.** Same posture as settings.local.json:
   per-install, machine-local. Teams that want it committed copy it
   out deliberately.

## Skill content (the canonical text this issue ships)

Keep it SHORT — it loads into agent context. Required substance:

- Frontmatter description that triggers on running tests/builds/
  verifiable commands in a repo with coreartifact installed.
- The recipe: run verifiable commands through
  `cart check <name> -- <cmd>` (exit codes pass through). In a
  headless/delegated session, self-bind:
  read the session id from the spool's last SessionStart line
  (`grep '"hook_event_name":"SessionStart"' .coreartifact/spool.jsonl | tail -1 | jq -r '.event.session_id'`)
  and pass `--session <id>`. That is what makes the session VERIFIED
  instead of unverified in the dashboard.
- The don'ts: never edit `.coreartifact/**`; never parse the spool or
  ledger directly (use `cart log` / `cart show`); never guess or
  fabricate a session id (binding is validated and will refuse).
- The reading: `‹absent›` means unverifiable-from-evidence, not
  missing — never "fix" it by inventing values.

## Boundaries

- The skill file is static text — no templating, no per-repo
  interpolation (byte-identical everywhere, which is what makes the
  drift check trivial).
- Do NOT touch the hook artifact, capture path, or ingest. This is
  install-surface only.
- Follow the existing settings-merge discipline for directory
  creation: a pre-existing `.claude/skills/` directory is entered,
  never clobbered.
- Read docs/gotchas.md before writing anything; the ISS-0022
  uninstall acceptance patterns are the model for the inversion test.
