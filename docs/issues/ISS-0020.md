# The drift detector — ABSENT always names its why

**Read `docs/gotchas.md` first** — entry 5 is this issue's entire reason to
exist: absent is never fabricated, and now it is never anonymous either.

The skeleton's thesis violated on itself: a facet could go ABSENT and
nothing named why. This issue makes ingest record an absence reason (through
the absence-record contract module — its enumerated constants, exactly)
whenever it degrades a session-level facet, with `kind` as the facet it
newly detects. Cost absences are emitted by the enrichment slice through the
same contract; this issue owns the detector's shape and the `kind` rules.

## The kind reconciliation (operator-ruled 2026-07-16 — build exactly this)

The naive reading "model stripped → kind ABSENT" would contradict the
fixture-verified v1 signal (`model` absent IS how headless presents) and
regress every real headless session. The accepted ruling consumes the second
recorded signal — SessionEnd `reason` — as **demote-only corroboration**: it
never classifies a session, it only refuses a contradicted classification.
The classification ladder:

1. SessionStart present with `model` → `interactive` (v1, unchanged).
2. SessionStart present without `model`, end-signal consistent (SessionEnd
   `reason` is `other`, or no SessionEnd at all) → `headless` (v1, unchanged
   — real headless, including every crashed session).
3. SessionStart present without `model`, **contradicted** by the
   interactive-only end-reason `prompt_input_exit` → `kind` ABSENT plus
   absence reason `model absent, contradicted by end-reason`.
4. No SessionStart line at all → `kind` ABSENT plus absence reason
   `no SessionStart captured`.

The recorded facts this rests on (from the recording pass): a clean
interactive `/exit` yields SessionEnd `reason: "prompt_input_exit"`;
headless clean completion yields `other`; crashed sessions may have no
SessionEnd at all, so the end-reason is useless there — which is why rule 2
treats "no SessionEnd" as consistent, never contradictory.

## Detector shape

Runs inside ingest's derived-projection recompute, per touched session,
after the facet derivations: for each participating facet (`kind` here;
`cost` via enrichment), either the facet is present — clear any absence
row — or it is ABSENT and the detector records the enumerated reason.
Idempotent: recompute converges, re-run changes no row counts. Because the
reasons derive purely from the verbatim spool (and transcript-at-path for
cost), delete-ledger + re-ingest rebuilds equivalent absence rows —
absences survive rebuild by construction, and the seam asserts it.

## Invariants (contract prose, not acceptance)

- Demote-only means demote-only: a session with `model` present classifies
  `interactive` regardless of its end-reason, and a stripped-model stream
  whose end-reason is `other` still classifies `headless` with no absence
  row — end-reason alone never classifies anything, and no new heuristic
  enters `kind` (not `source`, not `effort`, not UserPromptSubmit presence —
  all previously proven false signals).
- Facets outside the participating set (test-results, command outcome, the
  nullable sha/worktree columns) never emit absence rows — their ABSENT is
  already structural.
- The spool is never mutated; the hook artifact is byte-unchanged.

## Test-harness contract

Copy `tests/acceptance/harness/` verbatim; acceptance at
`tests/acceptance/ISS-0020/` drives the built CLI as a subprocess with
fixture replay through the installed hook command, asserting absence rows
through the absence-record contract's read API (imported from its owned
module — never raw SQLite, never a harness edit) and `kind` via the
harness's existing session reader.

Specifics the test-author must not go discover:

- **The drift stream is derived in the test, not committed**: load the
  interactive scenario through the typed loader, and strip the `model` key
  from its SessionStart payload in memory before replay. Its SessionEnd
  (`reason: "prompt_input_exit"`) survives and supplies the contradiction.
  Do not commit a hand-authored stream and do not touch the manifest —
  `corrupt-line.jsonl` stays the corpus's only committed hand-authored file.
- The non-regression control replays the unmodified headless scenario in
  the same test: `kind = headless`, zero absence rows for that session.
- The no-SessionStart case replays a loaded stream with its SessionStart
  line dropped in memory.
- Rebuild: snapshot absences, delete the ledger, run `log`, compare.
- Unit tests in the owned drift test file cover the classification ladder
  as pure logic: all four rules, plus SIGKILL-shaped input (no SessionEnd)
  landing in rule 2, never rule 3.
