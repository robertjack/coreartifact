# Kind classifier: demote-only on non-`startup` SessionStart sources

**Context (docs/recording-pass.md, findings 3 and 9 — read both before
writing anything).** The kind signal — `model` present on SessionStart →
interactive, absent → headless — was fixture-verified only at
`source: "startup"`. The PRD-0003 recording pass (2026-07-17) recorded a
third start mode live: an interactive session begun via `/clear` carries
`source: "clear"` and NO `model` key, so today's classifier fabricates
`headless` for it — the degradation law violated, and the PRD-0003 KPI
denominator polluted. The closed four-cell matrix is in finding 9;
operator ruling same day: **demote-only on non-startup sources**.

## The rule (three branches, exactly)

1. `model` present → `interactive`. Positive evidence, any source —
   unchanged.
2. `model` absent AND `source === "startup"` → `headless`. The verified
   cell — unchanged.
3. `model` absent AND anything else (`source` is a different value, or
   the key is missing) → kind **ABSENT** with an absence reason naming
   the observed source mode. Never classify an unobserved start mode
   from n=1 — `"clear"` may join a verified interactive-source set only
   via a future recording pass, not this issue.

## Contract (invariants — reviewer prose, not criteria; a faithful test
for these is green at base)

- Branch 1 and branch 2 behavior is byte-for-byte today's behavior; every
  existing kind test stays green untouched except where a locked test
  asserts the OLD branch-3 behavior (there is none — branch 3 is new
  territory).
- The existing `prompt_input_exit` demote-only corroboration
  (`src/ingest/drift.ts`, `CONTRADICTING_END_REASON`) is unchanged and
  still applies within startup-mode streams. The new rule is a SECOND
  demote gate in the same layer — mirror its shape and its absence-reason
  plumbing; do not invent a parallel mechanism.
- ABSENT is recorded via the existing absence-record contract
  (`src/core/absence.ts`) — a new `KIND_ABSENCE_REASONS` member plus its
  entry in the reason-validation set; doctor and the absences table pick
  it up through the existing plumbing, no new wiring.
- The hook artifact, the spool, the ledger schema (v2), and the
  registry are untouched. This is ingest-fold logic only.
