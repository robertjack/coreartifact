# coreartifact check — evidence badges through the spool

**Read `docs/gotchas.md` before writing anything** — entry 1 (no entry-point
guards), entry 3 (allowlist env), entry 4 (tests that cannot fail), entry 5
(the degradation law).

`coreartifact check <name> -- <cmd>` turns a named command run into
spool-borne evidence. The check envelope variant, the `checks` table and its
constraints are already settled (the check-variant contract issue — import
its parse/serialize from the core envelope module and the ledger schema from
the core ledger module; reinvent neither). This issue ships the CLI command,
the binding resolution, the output cap, and the ingest routing that projects
check lines into check rows.

## Command behavior

- Argv shape: `coreartifact check <name> -- <cmd> [args...]`. Everything
  after `--` is the wrapped command, run as a child process from the current
  directory with combined stdout+stderr captured. Missing `<name>`, missing
  `--`, or an empty wrapped command → usage error, exit nonzero, nothing
  written anywhere.
- The wrapped command's output streams through to the user as it runs
  (check wraps, it does not swallow), and is simultaneously captured for the
  spool line.
- Exit code: the CLI exits with the wrapped command's exit code, exactly. A
  failing command records a failing check — recording is never conditional
  on success. A wrapped command killed by a signal records the conventional
  128+signal exit code.
- **The spool is the only write path.** The command appends exactly one
  check line (via the core serializer, atomic O_APPEND — same discipline as
  capture) and never writes the ledger directly. Check rows exist only as
  ingest's projection, which is what makes R2 rebuild-equivalence hold.

## The output cap

`CHECK_OUTPUT_CAP_BYTES = 32768` (32 KiB), a named exported constant. If the
captured output's UTF-8 byte length exceeds the cap, store the first 32768
bytes truncated on a codepoint boundary (never split a multi-byte character)
and set `truncated: true`; otherwise store it whole with `truncated: false`.
Head-capture is deliberate: deterministic, and a check's primary signal is
its exit code — the opening lines carry the banner and first error.
Truncation is always flagged, never silent.

## Binding (captured once, frozen in the spool line)

The load-bearing ruling: binding is resolved ONCE at check time against the
repo ledger's then-current open-session set, and written into the spool
line. Ingest later projects `session_id`/`bound_by` verbatim and never
re-resolves — a point-in-time judgement only the spool can freeze.

Because binding needs the open-session set, `check` is a **reader**: it
triggers the same lazy ingest `log` does before resolving (so a deleted
ledger, or spool lines newer than the last ingest, still yield the true open
set). Resolution rules, in order:

1. `--session <id>` given and the id exists in the ledger → bind,
   `bound_by: "explicit"`. Unknown id → exit nonzero naming the id, **no
   line written**.
2. Exactly one `open` session → bind to it, `bound_by: "single-open"`.
3. Zero or several open → standalone: `session_id: null, bound_by: null` —
   never a guess (the honest-N/A shape). Fleets use `--session`.

Attribution of "the repo's ledger" is the existing core attribution
function — a worktree cwd resolves to the main checkout's spool and ledger;
a non-git cwd uses the init-root fallback.

## Ingest routing

Extend ingest's line classification (the touched ingest surface): a parsed
`v: 1` line routes by discriminator — `event` → the existing path, `check` →
`INSERT INTO checks ... ON CONFLICT(line_no) DO NOTHING`, projecting every
field including `session_id`/`bound_by` verbatim from the frozen line.
Neither-or-both discriminators → the existing corrupt-line handling (skip,
count, name, occupy the ordinal). Check lines consume `line_no` ordinals
from the same sequence as event lines. Idempotency and the HWM cursor are
untouched — a check line advances `lines_seen`/`ingested_bytes` exactly like
any physical line.

## Invariants (contract prose, not acceptance)

- The hook artifact is byte-unchanged; checks ride the CLI only.
- The spool is never mutated — check appends, ingest reads.
- pass/fail is derived from `exit_code == 0` at render time; no boolean is
  stored (the exit code is richer — 1 vs 137).
- Rendering checks in `log`/`show` is a later slice's work; this issue's
  proof is ledger rows, not output format.

## Test-harness contract

Copy `tests/acceptance/harness/` **verbatim** (tmpdir-repo factory with
isolated HOME and registry override, CLI runner, fixture replayer, ledger
readers). Acceptance lives at `tests/acceptance/ISS-0017/` and drives the
built CLI as a subprocess in tmpdir repos. Read check rows through the
check-variant contract's exported ledger row reader — import it from that
issue's ledger module; never raw SQLite in the test, and never an edit to
the acceptance harness (its files are outside this footprint).

Specifics the test-author must not go discover:

- **Making a session open:** replay the SIGKILL scenario via the typed
  loader — it has no SessionEnd and replayed timestamps are fresh, so after
  ingest it is `open`. The headless scenario ends with SessionEnd →
  `closed-clean`. One open session = SIGKILL alone; several open = SIGKILL
  plus a second no-SessionEnd replay with a distinct session id (the
  replayer's parallel/second-stream support); zero open = headless alone.
- `--session` takes a full session id from the replayed stream's payloads.
- The cap test wraps a command that prints more than 32 KiB including a
  multi-byte character straddling the boundary (e.g. node -e printing 'é'
  repeated); assert stored bytes, the codepoint-clean tail, and the flag.
  The under-cap case asserts byte-identical output and truncated false.
- Exit passthrough: wrap `node -e "process.exit(7)"` and assert exit 7 while
  the check line still landed.
- R2: run checks, snapshot check rows, delete the ledger file, run `log`,
  compare rows for equivalence (same line_no, same fields).
- Unit tests under `tests/unit/check/` cover pure logic only: cap
  truncation on codepoint boundaries, argv parsing, binding-rule selection
  over an in-memory open-session set.
