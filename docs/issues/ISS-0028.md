# Overview endpoint + classification/window (the verified-delegation headline)

`GET /api/overview[?repo=<root>]` — the cross-repo union view whose headline
IS the product's claim ("of the work I delegated, how much came back with
proof?"). High risk because the ABSENT-aware aggregation is subtle and the
correctness of the KPI is what the launch demo asserts. The binding shape,
field names, error codes, and SQL semantics are `api.md` Surface C — treat
them as fixed.

## Contract

`src/dashboard/overview.ts` is the handler wired into `src/dashboard/routes.ts`
(replacing the ISS-0027 stub). It performs the cross-repo union in application
code via `walkRegisteredRepos` (`src/resolve-session.ts`) — the same helper
`log`/`show` use, which **ingests-on-read** each reachable repo first
(Surface E: the dashboard is never staler than the spool). It never opens a
second ingest implementation and never a cross-database SQL join (each ledger
is its own SQLite file). `?repo=<root>` scopes the walk to one registered root
(unregistered → `404 repo_not_registered`).

`src/dashboard/classify.ts` holds the **pure** classification and window math
(no I/O) — the unit-test surface below the seam.

### The window

Anchored to the server host's local clock at request time. `end` = the request
instant; `start` = `end − OVERVIEW_WINDOW_DAYS` (a rolling 7×24h span). Emit
`window.start`/`window.end` as ISO-8601 with the server's local UTC offset
(the operator's wall-clock); `days` echoes the constant. The **SQL predicate**
compares against the **UTC-`Z`** forms of the same two instants, because
`sessions.started_at` is stored UTC-`Z` throughout the codebase and
lexicographic compare is only chronological when both sides share the zone
(api.md SQL semantics; PRD Amendment 1 fixture constraint).

### The three-way classification (api.md, CONTEXT.md — canonical)

KPI universe = **`kind = 'headless'` sessions in the window** (a delegated
session is an agent-run one; interactive is a human at the keyboard, excluded
from the KPI). Per headless session, over its bound checks
(`checks WHERE session_id = this session`):

- verified = at least one bound check AND zero failing bound checks
- failing = at least one failing bound check (failing = `exit_code <> 0`)
- unverified = zero bound checks

`verified + failing + unverified` partition `delegated_total`.
`unknown_kind` = count of in-window sessions with `kind` ABSENT (NULL) —
excluded from the denominator, surfaced separately, never folded into either
side. The API returns **counts only**, never a ratio (a zero denominator is
just `delegated_total: 0` with real zeros — R6, no server-side NaN).

### Tiles (all scoped to the same window, over all in-window sessions)

- `spend_present_usd` = SUM of present (`cost_usd IS NOT NULL`) costs — a
  derived aggregate.
- `cost_absent_count` = count of in-window sessions with `cost_usd` ABSENT.
  The anti-silent-zero guard: the two figures ship together, always.
- `sessions_by_kind` = `{ headless, interactive, unknown }` (unknown = kind
  ABSENT).
- `failing_checks` = count of **bound** checks with `exit_code <> 0` on
  in-window sessions (standalone checks, `session_id IS NULL`, never counted).

### sessions, repos, drift

- `sessions.latest` = in-window sessions, newest `started_at` first, capped at
  `LATEST_SESSIONS_LIMIT` **after** the merged cross-repo re-sort; `total` =
  the true in-window count across the union. Each entry carries `session_id`,
  `repo_root`, `kind`, `status`, `started_at`, `classification`
  (verified/failing/unverified for headless, else `null`), and `cost` as
  `{ value, derived: true }` (the derived-labeling rule, api.md B2).
- `repos` = one entry per registered root, **never silently skipped**: readable
  → `{ root, status: "ok" }`; unreadable → `{ root, status: "unreadable",
  reason }` (the `walkRegisteredRepos` failure reason). `repos_skipped` =
  corrupt registry lines the fold dropped (`FoldedRegistry.skipped`). Empty
  registry → `repos: []`, `repos_skipped: 0`, all-zero KPI/tiles.
- `drift` = one entry per in-window session whose `cc_version` is **non-null**
  and outside `TESTED_CLAUDE_CODE_RANGE` (`src/doctor/version.ts` — reused, not
  re-declared; currently `{min: "2.1.208", max: "2.1.212"}`). Compare under
  **semver ordering** (parse dotted integers — never string-compare "2.1.9" vs
  "2.1.10"). A NULL `cc_version` never drifts (degradation law: positive
  evidence only). The server **never executes `claude`** — drift reads the
  recorded `cc_version` only. All kinds are eligible.

## Shared-surface amendment: busy_timeout on the read path (api.md Flag 1)

`walkRegisteredRepos` in `src/resolve-session.ts` opens each ledger with
`new DatabaseSync(paths.ledger, { readOnly: true })` **without** a
`busy_timeout` pragma today. R5 requires every read connection the API opens
to set `PRAGMA busy_timeout = READ_BUSY_TIMEOUT_MS` (5000) so a concurrent
writer never surfaces "database is locked." Amend the shared helper once, here,
to set the pragma on open. This changes no `log`/`show` output (it only makes a
read wait instead of erroring), so the "no CLI render changes" wall stands.
ISS-0029 depends on this issue and inherits the amended helper.

## Test-harness contract (seeding is real-path only — read this)

Copy `tests/acceptance/harness/` verbatim (import, never fork). No direct
ledger writes — the spool is the only way in, in tests as in production.

- **Sessions A–E, kinds.** Seed each session by replaying a stream through the
  installed hook (the harness `replayFixtures`/`replayLines` path — the hook
  stamps a fresh UTC-`Z` `ts` at replay, so replayed sessions are in-window).
  Kind is classified by `src/ingest/drift.ts` from the SessionStart payload —
  **note the mapping, it is not intuitive**: a SessionStart carrying a `model`
  key → **interactive**; no `model` with `source: "startup"` → **headless**; a
  `source` that is not `startup` (e.g. `"clear"`) or absent → kind **ABSENT**.
  Use the recorded `headless` and `interactive` manifest fixtures for A/B/C
  (headless) and D (interactive); use `tests/fixtures/clear-source.jsonl`
  (source `clear`, no `model`) for the kind-ABSENT session E.
- **Bound checks (A passing, B failing).** Leave each session open (a replayed
  stream with no SessionEnd), then run the **real** `check` CLI via `runCli`
  (`check <name> -- <cmd>`) against it — the single-open-session binding rule
  binds it (`bound_by: single-open`). Use a command that exits 0 for A's
  passing check and a command that exits non-zero for B's failing check. Prior
  art for real-`check` binding: `tests/acceptance/ISS-0017/check.test.ts`.
- **Present cost.** Cost is enrichment-derived from the transcript at the
  session's recorded `transcript_path`. A replayed fixture's recorded absolute
  path does not exist in the tmpdir, so cost reads ABSENT unless you
  materialize the transcript: use the fixtures' sanctioned
  `buildSubstitutedTranscript` (`tests/fixtures/transcriptReplay.ts`) with the
  `cost-headless` pair (its transcript prices to a present cost). Seed at least
  one present-cost session and one cost-ABSENT session so both spend figures are
  asserted non-trivially.
- **The 8-day-old exclusion.** The hook stamps `ts` at replay (real now), so an
  old session cannot be seeded through the hook. Seed it by appending a
  well-formed envelope line directly to the registered repo's spool with `ts`
  set to `now − 8 days` in **UTC-`Z`** ISO-8601 (the spool is the sanctioned
  input path; this is not a ledger write). Seed a companion `now − 1 day`
  session the same way to assert the in-window side. Keep every hand-authored
  `ts`/`started_at` UTC-`Z` (Amendment 1: the window predicate compares UTC-`Z`
  strings lexicographically; a non-`Z` offset sorts silently wrong).
- **Drift.** `cc_version` is the transcript's `version` field. Hand-author a
  transcript whose `version` is outside the range (e.g. `2.1.220`) and
  materialize it at the session's `transcript_path` (the
  `buildSubstitutedTranscript` pattern with a hand-authored transcript under
  `tests/fixtures/dashboard/`); assert its drift entry appears. Assert an
  in-range session (the `cost-headless` transcript is `2.1.211`) yields no
  entry, and a transcript-unavailable session (NULL cc_version) yields none.
- **Union / unreadable / empty.** Register two repos; make one ledger
  unreadable (write garbage bytes to its `.coreartifact/ledger.sqlite`, or the
  reachability-preserving corruption the harness supports) and assert the
  healthy repo still serves and the bad one is listed `unreadable` with a
  reason. Assert the empty-registry shape separately.
- **Unit tests** (`tests/unit/dashboard-classify.test.ts`) exercise
  `src/dashboard/classify.ts` directly over seeded in-memory inputs: the
  three-way partition, the window boundary (inclusive vs excluded), and the
  semver drift comparison ("2.1.9" vs "2.1.10" must not string-compare).

Prove every acceptance by mutation where a bug could hide (gotcha's standing
rule): revert the classification/window/aggregate under test and watch the
assertion go red.

## Invariants (reviewer prose, not acceptance)

No schema change — every field derives from the v2 tables + registry + the two
reused code constants. The server never reads a transcript beyond what ingest
already did, never executes `claude`, never mutates the spool. The API never
fabricates an ABSENT value as `0`/`""`/`false`/`[]`. `log`/`show` output is
unchanged.
