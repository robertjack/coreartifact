# Core contracts: envelope, ledger, registry, attribution, status, CLI skeleton

This is the contract issue for coreartifact's walking skeleton. Every later
slice builds against the interfaces defined here; nothing here is allowed to
depend on a later slice. It is `high` risk because it fixes the persistent
data contract — the spool envelope and the registry are the two surfaces the
ledger cannot regenerate, and reshaping them after fan-out is the most
expensive change in the campaign.

## Vocabulary

Terms of art are exactly the project's canonical vocabulary: **spool**
(per-repo append-only JSONL, ground truth, never mutated by ingest, never
rotated), **envelope** (the spool line wrapper), **ledger** (per-repo SQLite,
a disposable rebuildable projection of the spool), **registry** (the single
global JSON list of ledger roots), **facet** (a derived evidence column;
ABSENT when its source was unavailable), **attribution** (resolving a session
to its ledger), **status**, **kind**, **footprint**, **agent_id** (the
canonical nesting key — there is no `subagent_id`).

## Modules this issue defines

Five seams, each independently testable below the acceptance seam with plain
vitest unit tests:

1. **Envelope** — parse and serialize spool lines.
2. **Ledger** — SQLite schema v1 DDL, open/create, typed row shapes.
3. **Registry** — read/write the global JSON list of ledger roots.
4. **Paths** — where the spool, ledger, hook artifact and registry live.
5. **Pure derivations** — attribution resolution and status derivation.

Plus a CLI program skeleton: an argv dispatcher exposing `init`, `log` and
`show` as registered commands. Each command's implementation lands in its own
later issue; this issue ships the dispatcher, usage output and exit codes,
and stub command handlers that exit nonzero with "not implemented". The
dispatcher module is a **shared surface**: later issues register their command
by touching it. Keep registration a one-line table entry so those amendments
never conflict.

## The envelope (spool line, v1)

One line per hook invocation, written with a single atomic `O_APPEND`, never
rewritten:

```
{ "v": 1, "ts": "<iso8601>", "event": <hook payload, verbatim> }
```

Boundary lines (SessionStart, SessionEnd) additionally carry a top-level
`git` sibling of `event`:

```
{ "v": 1, "ts": "<iso8601>", "event": <payload>,
  "git": { "head": "<sha>", "dirty": <bool> } }
```

`git.head` and `git.dirty` each follow the degradation law: the key is
present with its value, or ABSENT (key omitted or null) when git resolution
failed — a non-git cwd, or a repo with no commits. Never an empty string,
never a fabricated value.

**Byte preservation is a contract, not an optimization.** `parseEnvelope`
must hand back the `event` member's *source text*, not a re-serialized
object: re-serializing reorders keys and violates the "payload stored
verbatim, never rewritten" invariant that ingest depends on. The parser
therefore returns both the decoded envelope fields (`v`, `ts`, optional
`git`) and the raw `event` slice as a string. Design the return type so a
caller cannot accidentally round-trip the payload through `JSON.stringify`.

Parsing is total: it returns a typed result (`ok` with the parsed envelope, or
`err` with a reason) and never throws. Ingest relies on this to skip a corrupt
line and keep going.

### Contract shape chosen, and the shape rejected

Two shapes were designed for the envelope's payload handoff.

*Rejected:* a fully decoded `event: unknown` object, with ingest re-stringifying
it on insert. Ergonomic for consumers (nesting keys read straight off the
object) but it silently breaks byte preservation — the stored payload is a
re-serialization, key order drifts, and the "never rewritten" invariant is
unverifiable after the fact.

*Chosen:* a hybrid — the parser returns the raw payload text **plus** a decoded
view of it for field promotion. Consumers read nesting keys off the decoded
view; ingest stores the raw text. Byte preservation is structural, not a
discipline consumers must remember.

## The ledger (SQLite, schema v1)

The ledger lives at `<repo_root>/.coreartifact/ledger.db`, alongside the
spool. It is a **disposable, rebuildable projection** — deleting it and
re-ingesting the spool must rebuild equivalent rows. The DDL is fixed:

```sql
CREATE TABLE meta (
  id             INTEGER PRIMARY KEY CHECK (id = 1),
  schema_version INTEGER NOT NULL,
  ingested_bytes INTEGER NOT NULL DEFAULT 0,
  lines_seen     INTEGER NOT NULL DEFAULT 0,
  last_ingest_at TEXT
);

CREATE TABLE sessions (
  session_id     TEXT PRIMARY KEY NOT NULL,
  repo_root      TEXT NOT NULL,
  worktree_path  TEXT,
  kind           TEXT CHECK (kind IN ('headless','interactive')),
  status         TEXT NOT NULL
                 CHECK (status IN ('open','closed-clean','closed-inferred')),
  sha_before     TEXT,
  sha_after      TEXT,
  started_at     TEXT NOT NULL,
  last_event_at  TEXT NOT NULL,
  ended_at       TEXT
);

CREATE TABLE events (
  line_no         INTEGER PRIMARY KEY,
  session_id      TEXT NOT NULL,
  seq             INTEGER NOT NULL,
  ts              TEXT NOT NULL,
  hook_event_name TEXT NOT NULL,
  prompt_id       TEXT,
  agent_id        TEXT,
  agent_type      TEXT,
  tool_use_id     TEXT,
  payload         TEXT NOT NULL
);

CREATE TABLE footprint (
  session_id TEXT NOT NULL,
  path       TEXT NOT NULL,
  PRIMARY KEY (session_id, path)
);

CREATE INDEX idx_events_session ON events (session_id, seq);
```

No other tables and no other indexes. `meta.schema_version` is `1`.

`events.line_no` is the 1-based ordinal of the source line in the spool: the
event's identity, the `ON CONFLICT` dedupe anchor, and the reason a rebuild
produces equivalent rows. `seq` is a per-session presentation ordinal assigned
in `line_no` order.

**NULL means ABSENT.** This is the degradation law encoded structurally: a
facet whose source was unavailable is NULL, never an empty string, never zero,
never a plausible default. This module must make fabrication a schema
violation — hence the `CHECK` constraints on `status` and `kind`.

The SQLite driver choice is the implementer's (`node:sqlite` or a native
driver); the DDL above is vanilla `CREATE TABLE` and must be portable across
either. If a runtime dependency is added, `package.json` **and**
`pnpm-lock.yaml` change together.

## The registry (global JSON)

One global file at `~/.coreartifact/registry`:

```json
{ "v": 1, "ledgers": [ { "repo_root": "/abs/path", "added_at": "<iso8601>" } ] }
```

Entries are unique by `repo_root` — adding an already-registered root is a
no-op, which is what makes `init` idempotent. Reading a missing registry
returns an empty list, never an error. Writes are atomic (write-temp, rename)
so a concurrent reader never sees a half-written file.

The registry path must be overridable by an environment variable so the
acceptance harness can point a subprocess at a tmpdir registry instead of the
operator's real home. Name that variable in the paths module and treat it as
part of the contract — every later acceptance test depends on it.

## Attribution (pure)

Given a cwd and an init-root fallback, resolve the ledger:

- cwd inside a git **worktree** → repo root is the *main* repo root (via the
  git common dir), and the worktree checkout path is recorded alongside it.
- cwd inside a main checkout → that root, worktree path ABSENT.
- non-git cwd → the supplied init root fallback, worktree path ABSENT.

Capture must exist before attribution matters: this function is used by both
the hook artifact (to find the spool to append to) and ingest (to fill
`sessions.repo_root` / `worktree_path`), so it must be expressible without any
package dependency — pure `git rev-parse` shell-out plus path logic.

## Status derivation (pure)

`status` is a pure function of (end timestamp present?, last event timestamp,
now):

- **closed-clean** — a SessionEnd was captured.
- **closed-inferred** — no SessionEnd, last event older than the staleness
  threshold, a named exported constant of **12 hours**.
- **open** — no SessionEnd, last event inside the threshold.

Never a one-way door: it is recomputed on every ingest from the current facts,
so a late-ingested SessionEnd flips `closed-inferred` back to `closed-clean`.
SessionEnd `reason` does **not** discriminate (observed: `"other"` for both
clean completion and SIGTERM) — presence or absence of the SessionEnd event is
the only signal.

## Invariants (contract prose — reviewer's checklist, not acceptance)

These hold today and must still hold after this issue; they are not deltas and
must never appear as acceptance criteria:

- Payloads are stored verbatim and never rewritten.
- Transcripts are referenced by the path already inside the payload, never
  copied; no transcript column exists.
- Nothing leaves the machine — no network calls of any kind.
- No write path to the ledger or spool exists outside the CLI and the hook
  artifact.
- `package.json` keeps `private: true`.

## Test-harness contract

Unit tests only (plain vitest, per the stack profile) under
`tests/unit/core/`. Pure logic only: envelope parsing, attribution resolution,
status derivation, registry read/write, ledger open/create. There is no
acceptance harness yet — it lands in the harness issue and depends on this
CLI skeleton building.

For ledger tests, create the database in the test's own tmpdir; never touch a
real `~/.coreartifact`. For attribution tests, create real git repos and real
`git worktree add` checkouts in a tmpdir — no mocked git.

The existing `tests/scaffold.test.ts` placeholder stays green through this
issue; the harness issue deletes it.
