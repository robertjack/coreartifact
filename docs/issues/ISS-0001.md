# Core contracts I: envelope, paths, status, CLI skeleton

**Scope narrowed 2026-07-14 (escalation amendment).** This issue originally
carried five modules plus the CLI; it burned two full attempt ladders and
escalated twice. The failures clustered in exactly two areas, which are now
their own issues: **persistence** (ledger + registry) is ISS-0010, and
**attribution** (git shell-outs and their edge cases) is ISS-0011. What
remains here is pure logic with no I/O and no git — the parts that were never
the problem.

Prior art: the escalated branches `escalated/ISS-0001-attempt-1` and
`escalated/ISS-0001-attempt-2` hold working implementations of all of this.
Reuse them; do not rebuild from zero. Fix only what the acceptance criteria

## Vocabulary

Terms of art are exactly the project's canonical vocabulary (`CONTEXT.md`):
**spool**, **envelope**, **ledger**, **registry**, **facet** (ABSENT when its
source was unavailable), **attribution**, **status**, **kind**, **footprint**,
**agent_id** (there is no `subagent_id`).

## The envelope (spool line, v1)

One line per hook invocation, written with a single atomic `O_APPEND`, never
rewritten:

```
{ "v": 1, "ts": "<iso8601>", "event": <hook payload, verbatim> }
```

Boundary lines (SessionStart, SessionEnd) additionally carry a top-level `git`
sibling of `event`:

```
{ "v": 1, "ts": "<iso8601>", "event": <payload>,
  "git": { "head": "<sha>", "dirty": <bool> } }
```

`git.head` and `git.dirty` each follow the degradation law: **the key is
present with its value, or ABSENT (key omitted)** when git resolution failed.
Never an empty string, never a fabricated `false`. This binds the *parser* as
well as the writer: a `git.head` of `""` or `null` arriving from a malformed
line must not be blind-cast into the typed contract as if it were a sha — that
would admit a fabricated value into the evidence spine (reviewer S2, attempt 2).

**Byte preservation is a contract, not an optimization.** `parseEnvelope`
returns the `event` member's *source text*, not a re-serialized object:
re-serializing reorders keys and violates the "payload stored verbatim, never
rewritten" invariant that ingest depends on. The chosen shape is a **hybrid** —
the parser returns the raw payload text **plus** a decoded view of it for field
promotion. Consumers read nesting keys off the decoded view; ingest stores the
raw text. Byte preservation is structural, not a discipline consumers must
remember.

Parsing is **total**: it returns a typed result (`ok` with the parsed envelope,
or `err` with a reason) and never throws. Ingest relies on this to skip a
corrupt line and keep going.

**Serialization is one line, always.** `serializeEnvelope` must emit exactly one
physical line even when the payload contains embedded newlines or control
characters — a multi-line or syntactically invalid record in the append-only
spool corrupts ground truth itself and desynchronizes every line ordinal after
it (reviewer S1, attempt 2). The line-count contract of the whole system rests
on this function.

## Status derivation (pure)

`status` is a pure function of (end timestamp present?, last event timestamp,
now):

- **closed-clean** — a SessionEnd was captured.
- **closed-inferred** — no SessionEnd, last event older than the staleness
  threshold, a named exported constant of **12 hours**.
- **open** — no SessionEnd, last event inside the threshold.

"Present" means **genuinely present**. An empty string or an `undefined`
`endedAt` is NOT a captured SessionEnd; treating it as one fabricates
`closed-clean` for a crashed session — a false receipt, and the exact failure
mode the product exists to prevent (reviewer S2/S3, both attempts).

Never a one-way door: status is recomputed on every ingest from current facts,
so a late-ingested SessionEnd flips `closed-inferred` back to `closed-clean`.
SessionEnd `reason` does **not** discriminate crash from clean completion
(observed: `"other"` for both clean headless completion and SIGTERM) — presence
or absence of the SessionEnd event is the only signal.

## Paths

Where the spool, ledger, hook artifact and registry live. The registry root
**must be overridable by a named environment variable** — the acceptance
harness points subprocesses at a tmpdir registry instead of the operator's real
home, and every later acceptance test depends on it. Name the variable here and
treat it as part of the contract.

## CLI skeleton

An argv dispatcher exposing `init`, `log` and `show` as registered commands.
Each command's implementation lands in its own later issue; this issue ships the
dispatcher, usage output and exit codes, and stub handlers that exit nonzero
with "not implemented". The dispatcher is a **shared surface**: later issues
register their command by touching it. Keep registration a one-line table entry
so those amendments never conflict.

## The @types/node situation (do not re-derive)

`@types/node` cannot be fetched in the build sandbox (no network). Attempt 1
hand-rolled `src/core/node-shim.d.ts`; the reviewer then found it **shadows**
the real `@types/node` and breaks typecheck for later slices that need
`process.on` or `Buffer`. Resolve this deliberately: if `@types/node` is present
in the lockfile's resolution, use it; if a shim is unavoidable, it must not
shadow or collide with the real types when they are installed. Whatever is
chosen, `pnpm run typecheck` must pass **and** stay passable for every later
slice.

## Invariants (contract prose — reviewer's checklist, not acceptance)

- Payloads are stored verbatim and never rewritten.
- Transcripts are referenced by the path already inside the payload, never
  copied; no transcript column exists.
- Nothing leaves the machine — no network calls of any kind.
- No write path to the ledger or spool exists outside the CLI and the hook
  artifact.
- `package.json` keeps `private: true`.
