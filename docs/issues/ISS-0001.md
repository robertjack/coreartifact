# Core contracts I: envelope, paths, status, CLI skeleton

**Scope narrowed 2026-07-14 (escalation amendment).** This issue originally
carried five modules plus the CLI; it burned two full attempt ladders and
escalated twice. The failures clustered in exactly two areas, which are now
their own issues: **persistence** (ledger + registry) is ISS-0010, and
**attribution** (git shell-outs and their edge cases) is ISS-0011. What
remains here is pure logic with no I/O and no git — the parts that were never
the problem.

Prior art: the escalated branches `escalated/ISS-0001-attempt-1` and
`escalated/ISS-0001-attempt-2` hold working implementations of all of this.
Reuse them; do not rebuild from zero. Fix only what the acceptance criteria
below name.

## Vocabulary

Terms of art are exactly the project's canonical vocabulary (`CONTEXT.md`):
**spool**, **envelope**, **ledger**, **registry**, **facet** (ABSENT when its
source was unavailable), **attribution**, **status**, **kind**, **footprint**,
**agent_id** (there is no `subagent_id`).

## The envelope (spool line, v1)

One line per hook invocation, written with a single atomic `O_APPEND`, never
rewritten:

```
{ "v": 1, "ts": "<iso8601>", "event": <hook payload, verbatim> }
```

Boundary lines (SessionStart, SessionEnd) additionally carry a top-level `git`
sibling of `event`:

```
{ "v": 1, "ts": "<iso8601>", "event": <payload>,
  "git": { "head": "<sha>", "dirty": <bool> } }
```

`git.head` and `git.dirty` each follow the degradation law: **the key is
present with its value, or ABSENT (key omitted)** when git resolution failed.
Never an empty string, never a fabricated `false`. This binds the *parser* as
well as the writer: a `git.head` of `""` or `null` arriving from a malformed
line must not be blind-cast into the typed contract as if it were a sha — that
would admit a fabricated value into the evidence spine (reviewer S2, attempt 2).

**Byte preservation is a contract, not an optimization.** `parseEnvelope`
returns the `event` member's *source text*, not a re-serialized object:
re-serializing reorders keys and violates the "payload stored verbatim, never
rewritten" invariant that ingest depends on. The chosen shape is a **hybrid** —
the parser returns the raw payload text **plus** a decoded view of it for field
promotion. Consumers read nesting keys off the decoded view; ingest stores the
raw text. Byte preservation is structural, not a discipline consumers must
remember.

Parsing is **total**: it returns a typed result (`ok` with the parsed envelope,
or `err` with a reason) and never throws. Ingest relies on this to skip a
corrupt line and keep going.

**Serialization is one line, always.** `serializeEnvelope` must emit exactly one
physical line even when the payload contains embedded newlines or control
characters — a multi-line or syntactically invalid record in the append-only
spool corrupts ground truth itself and desynchronizes every line ordinal after
it. The line-count contract of the whole system rests on this function.

**The trap, named — it has now defeated three attempts.** Do NOT accept the
payload as an untagged `event: unknown` that silently treats a *string* as
"already-serialized JSON text, embed verbatim". The hook artifact is forbidden
by law from parsing the payload, so the law-conformant caller hands over the raw
stdin **text** — and raw stdin carries a trailing newline. Embedded unvalidated,
that writes **two physical lines** into the append-only spool and desynchronizes
every subsequent `line_no` forever, with no error raised. Executed proof
(attempt 3): a payload string with one trailing `\n` produced a 2-line record; a
payload with an interior newline produced a record that is not even valid JSON;
an `undefined` payload emitted the literal text `undefined`.

Make it unrepresentable: **tag the two inputs.** Accept a decoded value
(`event: unknown`, serialized here) OR pre-serialized text (`eventText: string`)
as *distinct, explicit* fields — never one field whose meaning depends on its
runtime type. On the text path, **hard-validate before embedding**: it must
parse as JSON **and** contain no control characters; otherwise return a typed
failure. Guard the `JSON.stringify(x) === undefined` case.

The acceptance test must feed it adversarial payloads — trailing newline,
interior newline, control characters, `undefined` — not just a well-formed
object. "Exactly one line for **any** payload" is the criterion; a test that
only passes objects proves a test-shaped subset of it.

**Totality means totality, on BOTH paths.** `serializeEnvelope` is documented as
never throwing, and the *decoded-value* path still does: `JSON.stringify` does
not return `undefined` for a **BigInt**, a **circular reference**, or a throwing
`toJSON`/getter/proxy trap — it **throws**, and the `=== undefined` guard
catches none of them (executed, 2026-07-14). A BigInt is not hypothetical: any
later slice computing a duration or token count as one crashes the spool writer
instead of receiving a typed failure. Wrap the encode in `try`/`catch` and
return a typed failure. A function whose contract says "never throws" must be
tested with inputs that make a naive implementation throw.

**Round-trip means byte-identical.** `eventText` with leading or trailing
whitespace is accepted today (spaces are not control characters and `JSON.parse`
tolerates them) but comes back trimmed — the writer accepts bytes it cannot give
back. Trim before embedding, or reject padded input; do not accept-and-alter.

## Status derivation (pure)

`status` is a pure function of (end timestamp present?, last event timestamp,
now):

- **closed-clean** — a SessionEnd was captured.
- **closed-inferred** — no SessionEnd, last event older than the staleness
  threshold, a named exported constant of **12 hours**.
- **open** — no SessionEnd, last event inside the threshold.

"Present" means **genuinely present**. An empty string or an `undefined`
`endedAt` is NOT a captured SessionEnd; treating it as one fabricates
`closed-clean` for a crashed session — a false receipt, and the exact failure
mode the product exists to prevent.

**The same law binds the other branch (2026-07-14 finding).** An unparseable
timestamp must not fabricate *liveness* either. `new Date("garbage").getTime()`
is `NaN`, and `NaN > threshold` is `false`, so a naive implementation silently
falls through to `open` — reporting a session as live forever when its
timestamp was corrupt. Guard it: a non-finite `lastEventAt` or `now` derives
**`closed-inferred`** (the honest, non-optimistic state), never `open`. Both
directions of this function must fail toward "we don't know", never toward a
flattering claim.

Never a one-way door: status is recomputed on every ingest from current facts,
so a late-ingested SessionEnd flips `closed-inferred` back to `closed-clean`.
SessionEnd `reason` does **not** discriminate crash from clean completion
(observed: `"other"` for both clean headless completion and SIGTERM) — presence
or absence of the SessionEnd event is the only signal.

## Paths

Where the spool, ledger, hook artifact and registry live. The registry root
**must be overridable by a named environment variable** — the acceptance
harness points subprocesses at a tmpdir registry instead of the operator's real
home, and every later acceptance test depends on it. Name the variable here and
treat it as part of the contract.

**The registry is a JSONL log, and paths must say so** (2026-07-14 finding —
the module was defaulting `registryRoot` to `~/.coreartifact/registry`, which is
neither the directory nor the log). `schema.md` Surface 3 and `CONTEXT.md` are
normative: the log lives at **`~/.coreartifact/registry.jsonl`**. Default the
registry *root* to `~/.coreartifact` and export the log path explicitly, so
ISS-0010 cannot guess wrong. The module whose entire job is "where things live"
is the last one allowed to be ambiguous about it.

## CLI skeleton

An argv dispatcher exposing `init`, `log` and `show` as registered commands.
Each command's implementation lands in its own later issue; this issue ships the
dispatcher, usage output and exit codes, and stub handlers that exit nonzero
with "not implemented". The dispatcher is a **shared surface**: later issues
register their command by touching it. Keep registration a one-line table entry
so those amendments never conflict.

**Separate the module from the entry — do not write an entrypoint guard**
(2026-07-14, established the expensive way). Two failed shapes, both executed:

1. `src/cli/index.ts` called `main()` at module scope, and `main` calls
   `process.exit` on every path — so **importing the module killed the
   importing process**. Any later test importing the command table silently
   aborts its vitest worker.
2. The fix attempt added an entrypoint guard comparing `import.meta.url` to a
   hand-rolled `file://` URL of `process.argv[1]` (hand-rolled to dodge the
   `@types/node` gap). It made the **shipped `bin` silently no-op**: ESM
   realpaths `import.meta.url` but not `argv[1]`, and `node_modules/.bin/*` is
   a **symlink** — so the guard reads false, `main` never runs, and the CLI
   exits **0 printing nothing**, including for an unknown command. It also
   breaks without any symlink, because the hand-rolled `encodeURIComponent`
   escapes `@` while Node's `pathToFileURL` does not — and pnpm's store path is
   literally `coreartifact@0.0.0`. A false success is the worst possible bug in
   this product.

The correct shape has no guard, because it has nothing to guard:

- **`src/cli/index.ts` is a pure module.** It exports `main` **and the command
  table** (both are public surfaces later issues amend and later tests import).
  It never invokes anything at module scope. Import-safe by construction.
- **`src/cli/bin.ts` is the entry.** Its entire body imports `main` and calls
  it, `.catch()`ing so a handler's rejection becomes a nonzero exit rather than
  an unhandled rejection. It is only ever executed, never imported — so "am I
  the entry?" is a question nobody has to ask.
- **`bin` points at the compiled entry** (`dist/cli/bin.js`). No `cp`, no
  generated wrapper, no codegen in the build script.

An acceptance test must spawn the bin **through a symlink** and **from a path
containing `@`** — the two real install layouts. Testing only the direct path is
what let this ship green.

## The @types/node situation (do not re-derive)

`@types/node` cannot be fetched in the build sandbox (no network). Attempt 1
hand-rolled `src/core/node-shim.d.ts`; the reviewer then found it **shadows**
the real `@types/node` and breaks typecheck for later slices that need
`process.on` or `Buffer`. Resolve this deliberately: if `@types/node` is present
in the lockfile's resolution, use it; if a shim is unavoidable, it must not
shadow or collide with the real types when they are installed. Whatever is
chosen, `pnpm run typecheck` must pass **and** stay passable for every later
slice.

## Invariants (contract prose — reviewer's checklist, not acceptance)

- Payloads are stored verbatim and never rewritten.
- Transcripts are referenced by the path already inside the payload, never
  copied; no transcript column exists.
- Nothing leaves the machine — no network calls of any kind.
- No write path to the ledger or spool exists outside the CLI and the hook
  artifact.
- `package.json` keeps `private: true`.

## Test-harness contract

Unit tests only (plain vitest, per the stack profile) under `tests/unit/core/`.
Pure logic only: envelope parse/serialize, status derivation, paths.

Acceptance tests for **this issue's own criteria** belong at
`tests/acceptance/ISS-0001/` per the harness convention (corrected 2026-07-14 —
an earlier revision of this spec forbade this outright, which conflicted with
how test-authors actually work; the spec was wrong, not the tests). What you
must **not** do is modify, extend or depend on the fixtures/loader harness that
ISS-0002 merged, or build a second way to spawn the CLI — the shared acceptance
harness is ISS-0003's.

`"test": "pnpm build && vitest run"` is **correct and intended**: any criterion
that exercises the built CLI entry needs the build to have run. Keep it.

The CLI's **`bin` must point at the real compiled entry** (`dist/cli/index.js`).
Do not `cp dist/cli/index.js dist/cli.js` — relocating the module up a directory
breaks every relative import inside it, so the shipped bin dies the moment the
dispatcher imports anything from `src/core` (which `init`, `log` and `show` all
will). Today's tests only pass because the dispatcher imports nothing yet.

`pnpm test` must be green at HEAD when you finish — a red suite is not "someone
else's problem", it is a broken gate for every later slice.
