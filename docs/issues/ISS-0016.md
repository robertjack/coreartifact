# Fixtures wired for the loop (R13's routed remainder)

**Read `docs/gotchas.md` first** — especially entry 4 (test the artifact
that ships) and the standing rule "could this test fail if the bug were
present?".

The operator-assisted recording pass is **complete** (2026-07-15, Claude
Code 2.1.211; protocol and findings in `docs/recording-pass.md`): five
transcript pairs, three new streams, envelope oracles, the backgrounded
probe, and the `claude --version` shape are all committed. What remains —
and was deliberately routed here rather than done as a drive-by — is wiring
them into the typed access every later slice consumes. Regenerating the
manifest is its own routed change, never a drive-by. Risk is `med`, not the
sketch's `high`: the keyboard work that made this tier high is already done.

## What exists on disk (do not edit any of it)

- Three new streams: `tests/fixtures/cost-headless.jsonl` (14 events),
  `tests/fixtures/vitest.jsonl` (8), `tests/fixtures/background.jsonl` (16),
  all recorded on 2.1.211. Deliberately NOT yet in the typed manifest.
- `tests/fixtures/transcripts/` — five transcripts, three envelope oracle
  files, and its own raw `manifest.json` carrying pairs, models, versions,
  oracle values (total_cost_usd, token counts, distinct_requests), the
  vitest suite shape, the background probe facts, and the recorded
  `claude --version` output shape. This raw file is ground-truth metadata;
  the typed layer reads it, never rewrites it.
- Every committed fixture file is byte-verbatim as recorded and **stays**
  that way — the law for this corpus. `corrupt-line.jsonl` remains the one
  hand-authored stream and remains outside every manifest.

## The work

1. **Typed manifest gains the three streams.** Add cost-headless, vitest and
   background entries to `tests/fixtures/manifest.json` in the existing
   entry shape (scenario, file, claudeCodeVersion, ordered event names —
   derive the event lists from the streams themselves, do not guess).
   Extend the loader's scenario name type to the eight scenarios.
2. **Typed transcript-pair access.** The loader gains a typed reader over
   the transcripts manifest at its fixed path (the loader's fixed-path,
   no-discovery discipline — same as the stream manifest): per pair, the
   scenario, stream path, transcript path, Claude Code version, model,
   and the oracle (null for the two recovered pairs) with total_cost_usd
   and the four token classes (input, output, cache_read, cache_creation),
   plus distinct_requests where present. Expose the recorded
   claude-version output shape string too — the doctor issue's shim uses it.
3. **The sanctioned substitution — a fixtures-owned replay wrapper.** The
   substitution lives in a new module this issue owns,
   `tests/fixtures/transcriptReplay.ts`; the acceptance harness is NOT
   edited (it is PRD-0001's acceptance surface — outside this footprint,
   byte-unchanged). Given a transcript pair and a test tmpdir, the wrapper
   copies the transcript fixture into the tmpdir, rewrites
   `transcript_path` in each payload line to that copy (every other
   payload byte unchanged), and returns the substituted lines plus the
   tmpdir transcript path — usable whole or as a truncated prefix (the
   background-outcome slice needs the prefix). A convenience entry
   delivers those lines to a hook command through the harness's exported
   explicit-lines replay primitive (`replayLines`, imported — mirroring
   what Claude Code does on a real machine). Committed fixture files stay
   byte-verbatim (assert this). Replay without the wrapper is untouched —
   existing harness self-tests must stay green untouched.

## Invariants (contract prose, not acceptance)

- No committed recording is edited, re-recorded, re-serialized or
  reformatted by this issue; the only files that change are the typed
  layer this issue owns (manifest, loader, transcript-replay wrapper) and
  its tests. The acceptance harness (`tests/acceptance/harness/`,
  `tests/acceptance/harness.test.ts`) is outside this footprint and stays
  byte-unchanged — the wrapper composes its exported replay primitive by
  import only.
- The corrupt-line fixture stays outside the manifest and unreachable
  through the loader (a hand-authored stream must never carry a version
  stamp it was not recorded on).
- No network, no real Claude Code invocation — this issue only wires
  committed bytes.

## Test-harness contract

Extend the existing `tests/unit/fixtures.test.ts` (manifest/stream
validation — the four-field check runs over all eight streams now); the
wrapper's self-test lives there too, NOT in the harness self-test file
(`tests/acceptance/harness.test.ts` is outside this footprint — read it
for the pattern, never edit it). The self-test pattern to copy is the
harness's existing replayer test: replay into a stub command that records
its stdin, then assert delivery. For the substitution test, the stub records
payloads; assert `transcript_path` in every delivered payload points inside
the tmpdir, the tmpdir copy's bytes equal the committed transcript's, and
the committed files' bytes are unchanged after the run. Nothing here needs
the built CLI.
