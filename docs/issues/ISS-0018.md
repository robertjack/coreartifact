# Parser interface + the vitest parser (the only one)

**Read `docs/gotchas.md` first** — entry 5 (the degradation law) governs
every branch of this issue.

Test-output parsing becomes evidence: behind a pluggable interface at
ingest, exactly one parser (vitest) turns a recorded vitest run into a
test-results facet. This issue carries both the interface contract and its
sole implementation — merged deliberately, because the interface's only
consumers (the ingest recompute and the one parser) land here; a standalone
contract issue would settle an interface nobody else builds against. The
interface is the durable artifact; pytest is v1.1, gated on demand, and
there is no parser registry or config — the set is hardcoded.

## The interface (design considered twice)

```
parse(command, stdout, stderr, exit) -> TestResults | null
```

Pure, synchronous, ingest-side. `null` means "not mine / unparsable" and the
facet stays absent. `TestResults` carries passed/failed/skipped counts,
failed test names (an array, empty when failed is 0), and an optional
duration in milliseconds (absent when unextractable — never 0 as a stand-in).
Parsers never run on the hot path, never see the transcript, and never see
raw hook payloads.

*Rejected shape, recorded:* `parse(event) -> TestResults | null` — handing
parsers the verbatim payload and letting each self-extract. Rejected because
the two payload paths (below) are the fragile, undocumented part; baking
that knowledge into every future parser would multiply the register's most
string-brittle dependency. Chosen shape: **ingest owns payload extraction**
(where the command, stdout-equivalent text and exit live per event shape)
and parsers stay trivially unit-testable over plain captured text.

## The two payload paths (recording-pass finding 7 — treat as ground truth)

- A **passing** run lands in `PostToolUse` at `tool_response.stdout`: plain
  text, no ANSI, with stable summary lines of the shape
  ` Test Files  1 passed (1)` / `      Tests  2 passed (2)` /
  `   Duration  65ms (...)`.
- A **failing** run exits 1 and lands as **`PostToolUseFailure`, which
  carries NO `tool_response`**: the entire vitest output — summary lines AND
  failed test names — is embedded in the `error` string after the marker
  `"Exit code 1\n\n"`. Ingest's extraction strips that marker prefix and
  hands the remainder to the parser as the output text with exit 1.

The parser claims a command by its output shape (the vitest summary lines),
not by grepping the command string for "vitest" — a `pnpm test` alias must
still parse; a command whose output lacks the summary shape returns null.
Skip counts parse from the summary when present; zero tests reported
(`Tests  0 passed (0)` or vitest's no-tests summary) is a claimed run with
zero counts — a real zero, not absence.

## Ingest recompute and identity

Extend ingest's derived-projection recompute (the touched ingest surface):
for each touched session's command events, extract, run the hardcoded parser
list, and upsert `test_results` keyed on the command event's `line_no`
(`ON CONFLICT(line_no) DO NOTHING` / deterministic recompute — re-run
changes no row counts). Because the row derives purely from the verbatim
stored payload, delete-ledger + re-ingest re-runs the parser and rebuilds an
equivalent row: rebuild-equivalence for free. No absence record is ever
written for unclaimed commands — a non-test command is not a degraded facet
(row membership already encodes absence).

## Render

`show` gains a minimal test-results badge line per claimed command (counts,
failed names, duration) — enough to satisfy "rendered in show". The full
badge/column treatment, absent markers and `log` changes belong to the
render slice, which touches this output afterwards; keep the render seam
thin (a function over a `TestResults` row) so that slice amends without
rework.

## Invariants (contract prose, not acceptance)

- Parsers never run in the hook artifact; capture still parses nothing.
- The spool is never mutated; `test_results` holds no ground truth the
  stored payloads cannot regenerate.
- `failed_names` is a real empty array when failed is 0, never NULL.

## Render-surface handoff

This issue owns `src/render/show.ts` for the campaign and adds only the
test-results badge line; the cost slice and the render-depth slice edit
that file downstream and both depend on this issue — the shared render
surface is serialized, never edited in parallel.

## Test-harness contract

Copy `tests/acceptance/harness/` verbatim; acceptance at
`tests/acceptance/ISS-0018/` drives the built CLI as a subprocess. The
vitest scenario is loaded by name through the typed loader (the fixtures
issue added it — depend on that, do not hand-wire paths). The recorded
run's expected values are pinned in the transcripts manifest's suite
metadata: 2 files, 4 tests, 1 failed, failed name
`subtracts (deliberately red for the fixture)`. Replay → run `show` → assert
the facet via the check-variant contract's exported test-results row reader
(imported from the ledger module — never raw SQLite, never a harness edit)
AND the badge line in show's stdout. Rebuild: snapshot the row, delete the ledger, rerun, compare.

Unit tests under `tests/unit/parsers/` run the parser over output text
extracted from the committed fixture stream's payloads (read them through
the loader — do not paste vitest output by hand into the test where a
recorded string exists), covering: passing summary, failing summary with
names, zero-tests, non-vitest output → null, duration absent → null
duration. The ingest-side extraction (marker stripping, which event shapes
feed the parser) is unit-covered under `tests/unit/ingest/`.

## Escalation amendment (2026-07-16, operator lane)

Two structural blocks, both confirmed twice by attempts that changed no
code because none was wrong: (1) scope_change GRANTED — the footprint
omitted `src/cli/commands/show.ts`, but rendering the badge requires
wiring line_no through the events SELECT and joining test_results
there; `touches` now includes it (the decomposer put the render seam in
show.ts's render module without granting the command file that feeds
it). (2) test_dispute — R4's single-token line match resolved to the
UserPromptSubmit echo of the command string, not the command's rendered
line; the match now requires the `command:` render prefix. The parser
implementation itself passed two attempts' independent verification
unchanged.

**Review remediation + ruling (2026-07-16):** the branch's first
adversarial review (post-escalation) found a false green — first-match
summary parsing let a console-logged `Tests N passed` line pre-empt the
real trailing summary; the parser now anchors on the LAST `Test Files`
block with the `Tests` line adjacent (poison input proven red then
green; operator re-executed the probe: real summary wins, failed names
exact). The seq-vs-line_no identity gap is closed by a corrupt-line-
injection unit test (mutant keying on seq proven red). RULING: vitest's
`No test files found` output carries no summary shape, so it is NOT a
claimed run (null, no row) — claiming is by shape, and the command's
exit code records universally regardless; the zero-count row applies
only to outputs carrying the summary block (`Tests  no tests` verified
to store zeros).
